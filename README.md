# Документация для Аналитика

Репозиторий содержит веб-систему для составления опросов по модели Кано и анализа собранных ответов. В этой документации описан только сценарий работы Аналитика: создание опроса, контроль сбора ответов и интерпретация результатов анализа.

## Назначение системы

Система помогает Аналитику определить, как отдельные свойства продукта влияют на удовлетворенность пользователей. Для каждого свойства автоматически формируется пара вопросов по модели Кано:

- функциональный вопрос: как пользователь отнесется к наличию свойства;
- дисфункциональный вопрос: как пользователь отнесется к отсутствию свойства.

После сбора ответов система классифицирует свойства по категориям Кано, считает коэффициенты удовлетворенности и неудовлетворенности, формирует порядок приоритизации и строит матрицу Кано.

## Основные экраны Аналитика

### Главная страница

После входа Аналитик попадает на `/home`. Отсюда доступны основные рабочие разделы:

- конструктор опроса `/builder`;
- панель анализа `/dashboard`;
- панель анализа конкретного опроса `/dashboard/<survey_id>`.

### Конструктор опроса

Страница `/builder` предназначена для создания нового опроса.

Аналитик заполняет:

- название опроса;
- список свойств продукта;
- описание каждого свойства, если оно необходимо для уточнения смысла.

Для каждого свойства система назначает внутренний идентификатор вида `А1`, `А2`, `А3`.

После нажатия кнопки создания система возвращает:

- `survey_id` — уникальный идентификатор опроса;
- ссылку на страницу прохождения опроса;
- ссылку на панель анализа этого опроса.

Аналитику важно сохранить `survey_id` или сразу перейти в панель анализа, потому что по нему открываются результаты конкретного опроса.

### Панель анализа

Страница `/dashboard` используется для контроля созданных опросов и запуска анализа.

В панели доступны:

- список созданных опросов;
- количество собранных ответов по каждому опросу;
- переход к странице анализа конкретного опроса;
- ручная загрузка опроса по `survey_id`;
- запуск анализа;
- очистка ответов;
- удаление опроса.

На этой странице система показывает название опроса, количество свойств, количество собранных ответов и ссылку на страницу опроса.

## Рабочий сценарий Аналитика

1. Открыть конструктор `/builder`.
2. Ввести название продукта для исследования.
3. Добавить свойства продукта, которые нужно проверить.
4. Создать опрос.
5. Передать полученную ссылку на опрос целевой аудитории.
6. Открыть панель анализа созданного опроса.
7. Следить за количеством собранных ответов.
8. После накопления данных нажать `Провести анализ`.
9. Изучить категории Кано, коэффициенты и рекомендации.
10. Использовать блок приоритизации для принятия продуктовых решений.

## Как формулировать свойства

Свойство должно быть конкретным и понятным для аудитории. Хорошая формулировка описывает одну проверяемую возможность продукта.

Примеры удачных свойств:

- автоматическое сохранение черновика;
- экспорт отчета в PDF;
- уведомление о просроченной задаче;
- фильтрация результатов по подразделению.

Не стоит объединять несколько смыслов в одно свойство. Например, вместо `уведомления и отчеты` лучше создать два отдельных свойства: `уведомления` и `отчеты`.

## Шкала ответов

В системе используется стандартная пятибалльная шкала Кано:

- `1` — Мне нравится;
- `2` — Ожидаю;
- `3` — Нейтрально;
- `4` — Могу терпеть;
- `5` — Мне не нравится.

Каждый ответ на свойство состоит из двух оценок: реакции на наличие свойства и реакции на его отсутствие. На основе этой пары система определяет категорию Кано.

## Результаты анализа

После запуска анализа панель показывает сводку и карточки по каждому свойству.

### Категории Кано

- `M`, обязательная: базовое ожидание. Отсутствие свойства вызывает недовольство, наличие воспринимается как норма.
- `O`, одномерная: чем лучше реализовано свойство, тем выше удовлетворенность.
- `A`, привлекательная: свойство приятно удивляет и может стать конкурентным преимуществом.
- `I`, безразличная: свойство почти не влияет на восприятие продукта.
- `R`, обратная: наличие свойства может ухудшать восприятие.
- `Q`, сомнительная: ответы противоречивы или требуют проверки формулировки.

### Коэффициенты

В карточке свойства отображаются два коэффициента:

- коэффициент удовлетворенности показывает, насколько наличие свойства способно повысить положительное восприятие продукта;
- коэффициент неудовлетворенности показывает, насколько отсутствие свойства ухудшает опыт.

Коэффициент неудовлетворенности выводится отрицательным числом. Чем ближе он к `-1`, тем сильнее риск недовольства при отсутствии свойства.

### Распределение ответов

Для каждого свойства отображается количество попаданий в категории:

```text
A, O, M, I, R, Q
```

Это помогает понять, насколько устойчив итоговый вывод. Если итоговая категория победила с небольшим отрывом или много ответов попало в `Q`, формулировку свойства стоит проверить повторно.

### Порядок развития

Панель анализа формирует приоритеты:

1. сначала обязательные свойства `M`;
2. затем одномерные свойства `O`;
3. затем привлекательные свойства `A`;
4. свойства `I` получают низкий приоритет;
5. свойства `R` лучше пересмотреть или исключить;
6. свойства `Q` требуют повторной проверки.

## Матрица Кано

При запуске анализа система строит изображение матрицы Кано.

Если библиотека для работы с изображениями недоступна, анализ продолжит работать, но матрица не будет создана.

## API, используемое интерфейсом Аналитика

### Создание опроса

```http
POST /api/surveys
```

Тело запроса:

```json
{
  "title": "Опрос по личному кабинету",
  "features": [
    {
      "name": "Экспорт отчета в PDF",
      "description": "Возможность скачать сформированный отчет"
    }
  ]
}
```

Результат содержит `survey_id`, ссылку на опрос и ссылку на панель анализа.

### Список опросов

```http
GET /api/surveys
```

Возвращает список опросов, их идентификаторы, названия, количество свойств и количество собранных ответов.

### Получение данных опроса

```http
GET /api/survey?id=<survey_id>
```

Возвращает название, свойства и автоматически сформированную таблицу вопросов.

### Количество ответов

```http
GET /api/responses?id=<survey_id>
```

Возвращает число собранных ответов для выбранного опроса.

### Запуск анализа

```http
POST /api/analyze
```

Тело запроса:

```json
{
  "survey_id": "opros-20260423204233-5da9",
  "create_chart": true
}
```

Возвращает:

- сводку по количеству свойств;
- количество ответов;
- распределение свойств по категориям;
- порядок приоритизации;
- результаты по каждому свойству;
- предупреждения по качеству данных;
- ссылку на матрицу Кано, если она была создана.

### Очистка ответов

```http
POST /api/responses/clear
```

Тело запроса:

```json
{
  "survey_id": "opros-20260423204233-5da9"
}
```

Используется, если Аналитику нужно сбросить собранные данные и начать сбор заново.

### Удаление опроса

```http
POST /api/survey/delete
```

Тело запроса:

```json
{
  "survey_id": "opros-20260423204233-5da9"
}
```

Удаляет сам опрос и связанные с ним ответы.

## Рекомендации по интерпретации

Высокий приоритет стоит отдавать свойствам `M` и `O`: первые закрывают базовые ожидания, вторые напрямую повышают удовлетворенность. Свойства `A` полезны для усиления продукта, но обычно идут после обязательной базы. Свойства `I` не стоит реализовывать без дополнительного обоснования, а свойства `R` требуют осторожности, потому что часть аудитории может воспринимать их негативно.

Если свойство попало в `Q`, лучше проверить формулировку вопроса, убедиться, что свойство понятно аудитории, и при необходимости провести повторный опрос.
